# Contributing

Elev8 OS is currently a private project built for Elev8 Arts.

## Before changing code

Every change should begin with one of these:

- A documented problem
- A documented opportunity
- A documented bug

The item must explain:

- What is happening
- Why it matters
- Who benefits
- What success looks like
- Priority
- Dependencies

## Branch naming

Use clear branch names:

```text
feature/waitlist-engine
feature/ceo-dashboard
fix/artist-profile-save
fix/contact-link-format
docs/update-roadmap
```

## Commit messages

Use concise, action-oriented messages:

```text
Fix artist contact link normalization
Add waitlist entry data model
Document payout rounding rules
```

## Pull requests

Every pull request should include:

- Problem solved
- Summary of changes
- Testing performed
- Screenshots for UI changes
- Database changes
- Risks
- Rollback instructions

## Do not commit

- Passwords
- API keys
- Production database exports
- Customer personal information
- Tax documents
- Uploaded private files
- WordPress configuration secrets

## Definition of done

A change is done only when:

- Code is complete
- Tests pass
- Documentation is updated
- Release notes are written
- Staging verification is complete
