# Branching Guide

## Main branch

`main` must represent the latest approved, tested version.

Do not make experimental changes directly on `main`.

## Branch types

```text
feature/name
fix/name
docs/name
release/version
hotfix/name
```

Examples:

```text
feature/native-waitlist
fix/artist-profile-critical-error
docs/founders-edition-plan
release/5.0.0
hotfix/payout-rounding
```

## Beginner workflow in GitHub Desktop

1. Select **Current branch**.
2. Click **New branch**.
3. Enter the branch name.
4. Make changes.
5. Commit the changes.
6. Publish the branch.
7. Create a pull request on GitHub.
8. Review and test.
9. Merge into `main`.
