# UI Standards

## Design goals

Elev8 OS should feel clear, welcoming, practical, and calm.

It should not feel like a cluttered WordPress utility.

## Principles

- One obvious primary action per screen
- Plain language
- Avoid unnecessary technical terms
- Show important totals first
- Keep financial explanations visible
- Clearly separate public and private data
- Mobile-friendly artist pages
- Consistent spacing and typography
- Helpful empty states
- Clear success and error messages
- Never hide destructive actions inside ambiguous controls

## Naming

Use **Elev8 Member Artist** in user-facing language rather than Amelia employee.

Internal integration code may still use Amelia's terminology when necessary.

## Financial displays

Always show:

- Source amount
- Rule used
- Calculation
- Final Elev8 share
- Final artist share
- Refunds or adjustments

## Empty states

Do not display a blank box.

Explain what is missing and what the user should do next.
